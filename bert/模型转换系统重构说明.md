# 模型转换系统重构说明

## 📋 更新概览

本次更新对 TensorFlow 到 PyTorch 的模型转换系统进行了全面重构，提升了自动化程度和用户体验。

---

## ✨ 主要改进

### 1. 智能批量转换脚本

**文件**: `bert/convert_tf_to_pytorch.py`

#### 新功能特性

- ✅ **自动扫描**: 自动检测 `model/` 目录下所有包含 TensorFlow 格式的模型
- ✅ **智能去重**: 检测已转换的模型（存在 `pytorch_model.bin` 和 `config.json`），避免重复转换
- ✅ **多模型支持**: 自动识别 BERT 和 RoBERTa 模型类型
- ✅ **批量处理**: 一次性处理所有未转换的模型
- ✅ **详细报告**: 提供转换统计信息（成功/失败/跳过数量）

#### 使用方法

```bash
python bert/convert_tf_to_pytorch.py
```

### 2. 依赖优化

**文件**: `requirements.txt`

#### 变更内容

- ❌ **移除**: `tensorflow-cpu>=2.10.0`（不再是必需依赖）
- ✅ **新增**: `huggingface_hub>=0.20.0`（用于直接下载 PyTorch 模型）

#### 原因说明

1. **TensorFlow 非必需**: 
   - 新版 transformers 移除了内置的 TF→PyTorch 转换模块
   - 推荐直接从 Hugging Face 下载预转换好的 PyTorch 模型
   - 减少依赖复杂度，加快安装速度

2. **Hugging Face Hub**:
   - 提供命令行工具 `hf download` 直接下载 PyTorch 格式模型
   - 比手动转换更简单、更快速、更可靠

#### 安装命令

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 🔄 工作流程对比

### 之前的流程

```
1. 下载 TensorFlow 格式模型
2. 安装 tensorflow-cpu（~500MB）
3. 手动运行转换脚本（可能失败）
4. 检查是否生成 pytorch_model.bin
5. 如果失败，手动查找解决方案
```

### 现在的流程（推荐）

```
1. 直接使用 hf 下载 PyTorch 格式模型
   hf download hfl/chinese-bert-wwm-ext --local-dir model/xxx
   
或

1. 运行自动转换脚本
   python bert/convert_tf_to_pytorch.py
2. 如果自动转换失败，按照提示使用 hf 下载
```

**优势**:
- ⚡ 更快：下载比转换快得多
- 🎯 更可靠：官方预转换的模型保证可用
- 💾 更节省空间：无需安装 TensorFlow
- 🤖 更智能：自动检测，避免重复操作

## 🛠️ 技术实现细节

### 模型类型检测算法

```python
def detect_model_type(bert_config):
    num_hidden_layers = bert_config.get("num_hidden_layers", 12)
    hidden_size = bert_config.get("hidden_size", 768)
    
    # RoBERTa 通常层数更多或隐藏层更大
    is_roberta = num_hidden_layers > 12 or hidden_size > 768
    return "roberta" if is_roberta else "bert"
```

### 转换状态检测

```python
def check_model_converted(model_dir):
    """检查模型是否已经转换为 PyTorch 格式"""
    pytorch_model = os.path.join(model_dir, "pytorch_model.bin")
    config_json = os.path.join(model_dir, "config.json")
    
    # 必须同时存在 pytorch_model.bin 和 config.json
    return os.path.exists(pytorch_model) and os.path.exists(config_json)
```

### Hugging Face 配置生成

根据模型类型自动生成不同的配置：

**BERT 配置特点**:
- `directionality`: "bidi"
- `layer_norm_eps`: 1e-12
- `max_position_embeddings`: 512
- `pad_token_id`: 0

**RoBERTa 配置特点**:
- `bos_token_id`: 0
- `eos_token_id`: 2
- `layer_norm_eps`: 1e-5
- `max_position_embeddings`: 514
- `pad_token_id`: 1

## 🔍 故障排查

### Q1: 转换脚本报错 "RoBERTa转换模块不存在"

**A**: 这是预期行为。新版 transformers 移除了转换模块。

**解决方案**: 使用 `hf download` 直接下载 PyTorch 格式模型。

### Q2: 如何确认模型已正确转换？

**A**: 运行测试脚本：
```bash
python test/test_model_loading.py
```

或检查文件是否存在：
```bash
# 应该看到 pytorch_model.bin 和 config.json
dir model\chinese_bert_wwm_ext_L-12_H-768_A-12
```

### Q3: hf 命令找不到

**A**: 安装 huggingface_hub：
```bash
pip install huggingface_hub -i https://pypi.tuna.tsinghua.edu.cn/simple
```
